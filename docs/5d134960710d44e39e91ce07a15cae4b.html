GCP

How to Integrate GCP with Lema - 15 minutes
❗
To complete this integration, you will need: - Super Admin Permissions for Google Workspace - Google Cloud Platform (GCP) access - Lema user account
💡
NOTE: If you’ve already integrated Google Workspace with Lema, you can integrate GCP with just 1 click on the GCP integration page.
About This Integration:
GCP integration streamlines cloud operations, enhancing scalability, security, and performance, fostering seamless collaboration and unlocking advanced data management.
Integration Category: IDP, Cloud Infrastructure
Data Collected from This Integration: • Devices • Users • SaaS data
Integration Steps Overview:
Create a Service Account
Create Credentials for the Service Account
Set up a Domain-Wide Delegation for the Service Account
Enable an API for GCP
Connecting GCP’s APIs with Lema

1. Create a Service Account

❗
If you already have a Google Service Account you wish to use, skip to the next section.
Steps (5)
In the Google Cloud console, go to Menu > IAM & Admin > Service Accounts
Click Create Service Account
💡
A Service Account enables secure access to Google Workspace data without exposing user credentials.
Fill in the service account details, then click Create and Continue.
Filling the ‘Role’ field is optional. You may delete it.
Click Done.

2. Create Credentials for the Service Account

Steps (4)
Go to Service Accounts and select Lema’s service account from the table.
Click on the tab: Keys > Add key > Create new key.
Select JSON as the key type, then click Create.
Your new public/private key pair is generated and downloaded to your machine as a new file. Save the downloaded JSON file as credentials.json in your working directory. This file is the only copy of this key.

3. Set up Domain-Wide Delegation for a Service Account

Steps (8)
Go to Service Accounts and select Lema’s service account from the table.
Click Show advanced settings.
Under "Domain-wide delegation," find your service account's "Client ID." Click Copy to copy the client ID value to your clipboard.
💡
The following steps are relevant to the Google Workspace Admin Console and requires a Super Administrator User
In the Google Admin console, go to Menu menu > Security > Access and data control > API controls
Click Manage Domain Wide Delegation.
Click Add new.
In the "Client ID" field, paste the client ID that you previously copied.
In the "OAuth Scopes" field, enter a comma-delimited list of the scopes required:
https://www.googleapis.com/auth/cloud-platform

4. Enable an API for GCP

Steps (3)
In ‘API & Services’, enable:
Cloud Resource Manager API
Identity and Access Management (IAM) API

5. Connecting GCP’s APIs with Lema

Steps (2)
Lastly, to complete this integration, fill the 3 inputs on Lema’s GCP Integration Page (https://…lema.ai/gcp):
Name this integration (because you can have multiple integrations of the same tool).
Provide the email address associated with the super-admin account which you used to set up the domain-wide delegation.
Upload the credentials.json file (which you obtained in step 2;4).
💡
This information is what connects Lema to GCP.
Click Add Integration. Congratulations on your new integration! 🎉 It should appear in Lema’s Integrations table in Settings.
